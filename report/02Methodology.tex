\section{Methodology}

\subsection{Algorithms and Architecture}
The project requires the complete manual implementation of the forward sequence, performance evaluation (loss), and backward sequence (gradient traversal) for two distinct models.

\subsubsection{Multilayer Perceptron (MLP) Implementation}
The MLP translates a flattened 784-dimensional image vector through a hidden feature space of 128 neurons before compiling logits for the 10 digit classes. The forward sequence dictates:
\begin{equation}
Z^{[1]} = XW^{[1]} + B^{[1]}, \quad A^{[1]} = \sigma\left(Z^{[1]}\right) = \frac{1}{1 + e^{-Z^{[1]}}}
\end{equation}
\begin{equation}
Z^{[2]} = A^{[1]}W^{[2]} + B^{[2]}, \quad A^{[2]} = \text{softmax}\left(Z^{[2]}\right)
\end{equation}
During the backward pass (refer to Figure \ref{fig:mlp_flowchart}), gradients are propagated from the cross-entropy cost function using the chain rule. The explicit parameters update follows vanilla Gradient Descent logic, systematically penalizing weights based on their specific contribution to classification error averages across the batch.

\subsubsection{Convolutional Neural Network (CNN) Implementation}
Unlike the flattened MLP, the CNN natively parses the 2D configuration ($28\times 28$) of the image. The forward pass applies a valid-mode 2D convolution sweeping over the input pixels:
\begin{itemize}
    \item \textbf{Convolution Output}: A localized dot-product between a defined $K\times K$ kernel and the target image region, followed immediately by a ReLU activation ($A = \max(0, Z)$).
    \item \textbf{Fully Connected Sequence}: The localized feature maps are then flattened to intersect with a dense weight matrix resolving into Softmax distribution predictions. 
\end{itemize}

\subsection{Initialization and Numerical Stability}
To jumpstart training and prevent mathematical dead-ends, weights are carefully initialized using scaled random normal distributions (e.g., He or Xavier initializers). Special care is taken inside the Softmax implementation by subtracting maximum row values prior to exponentiation explicitly preventing `NaN` overflows, and a tiny $\epsilon$ is paired alongside logarithmic operations to mitigate division-by-zero bounds during loss evaluations.
